Resubmission Process

During resubmission, you can modify your dataset in several ways:

  • Add new files - Upload additional files to your dataset
  • Modify existing files - Replace existing files with updated versions
  • Remove files - Delete files from your dataset
  • Keep files unchanged - Leave existing files as they are

File States

In the resubmission interface, each file has a state that determines what action will be taken:

  • NONE - File will remain unchanged (default for existing files)
  • ADD - File will be added to the dataset (default for new files)
  • MODIFY - File will be replaced with a new version
  • DELETE - File will be removed from the dataset
Important: When you mark a file as MODIFY, you must upload a new file with the same name. The system will automatically replace the old file with the new one.

Validation

The resubmission process includes comprehensive validation:

  • Duplicate detection - Prevents adding files that already exist (unless marked as MODIFY)
  • Missing file validation - Ensures all files marked as MODIFY have corresponding new files
  • File integrity checks - Validates file formats and completeness
  • Metadata validation - Ensures all required metadata is provided
Validation Errors: If validation fails, you'll see detailed error messages with suggestions on how to fix the issues. All validation errors must be resolved before submission can proceed.
