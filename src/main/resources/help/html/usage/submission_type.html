Submission Type Selection

Previous | Next | See Also

ProteomeXchange Submission Tool assists you to submit your MS/MS datasets to multiple key proteomics data repositories, such as: PRIDE and PeptideAtlas.


Before starting the submission, you need to decide on which type of submission to make:
  • Complete submission
  • Partial submission
  • Affinity proteomics submission
You can select the type of submission by clicking on the checkboxes / button highlighted below:

Complete submission

This type is highly recommended. You will be issued with not only a ProteomeXchange accession but also a permanent DOI number to uniquely identify your submission in the future. With additional benefits that, PRIDE will show your data and allow private journal review support. Your data will also be available on the ProteomeXchange FTP to download once it has been made public.

The complete submission requires the following information:

  • Result Files: They are either mzIdentML or mzTab with spectrum files such as: MGF, mzML.
  • Raw Files: MS instrument outputs, such as: BRUKER .baf files. If your raw outputs are directories instead of files, please compress them into individual files before upload.
  • PRIDE Login: PRIDE user name and password. All submissions to the ProteomeXchange are private by default, the user name and password will help us to secure your submissions. You can register a new user account at PRIDE
  • Experiment Details: Details about the setup and context of your experiment (e.g., sample details, protocols, species, tissues)
  • Lab Head: The affiliation of the lab head (e.g., name, email, and address)

    • Although not required, we would also recommend you to submit the following data files:

  • Search result files: the original output from your search engine or your analysis pipeline, such as: Mascot .dat files. They should contain the peptide/protein identifications.

    • Partial submission

    This type is not recommended, since it will significantly reduce the visibility and reusability of your dataset. You should only choose this option if your identification results cannot be generated to the mzIdentML or mzTab format. Consequently, you will be only issued with a ProteomeXchange accession, and your dataset will only be available on the ProteomeXchange FTP to download.

  • Search result files: the original output from your search engine or your analysis pipeline, such as: Mascot .dat files. They should contain the peptide/protein identifications.
  • Raw Files: MS instrument outputs, such as: BRUKER .baf files. If your raw outputs are directories instead of files, please compress them into individual files before upload.
  • PRIDE Login: PRIDE user name and password. All submissions to the ProteomeXchange are private by default, the user name and password will help us to secure your submissions. You can register a new user account at PRIDE
  • Experiment Details: Details about the setup and context of your experiment (e.g., sample details, protocols, species, tissues)
  • Lab Head: The affiliation of the lab head (e.g., name, email, and address)

    • Affinity proteomics submission

    This submission type is specifically designed for affinity proteomics datasets. Affinity proteomics experiments involve the targeted quantification of proteins in a biological sample using specific binding interactions between molecules, such as antibodies and antigens (e.g., SomaScan or Olink experiments).

    A PAD identifier will be provided to uniquely identify the dataset.

    The affinity proteomics submission requires the following information:

  • Protein Quantification Files: Files containing protein quantification data from affinity proteomics experiments.
  • Raw Data Files (Recommended): Raw instrument outputs, if available.
  • PRIDE Login: PRIDE user name and password for secure submission.
  • Experiment Details: Details about the affinity proteomics experimental setup and context.
  • Lab Head: The affiliation of the lab head (e.g., name, email, and address).

    • Note: This submission type is specifically tailored for affinity proteomics workflows and may have different file type requirements compared to standard proteomics submissions.
