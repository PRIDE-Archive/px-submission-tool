What happens to my dataset?

An acknowledgement email will be sent to you, and we will deal with the rest. Once your datasets are processed by the ProteomeXchange, you will be notified with another email, it will include the information for you to view the result. If you have any question about your submission, you can also contact us by simply replying to the email you got from us.

Your dataset will be initially stored as private experiments in the PRIDE database. Once you instruct us to make them public, they will be propagated to other proteomics repositories such as PeptideAtlas.

Why my submission keeps failing?

There are several things you should check:

  • Network connection: Try using a wired connection if you're on wireless, as wireless networks can cause connection timeouts.
  • Firewall settings: Check with your system administrator to ensure connections to external servers are allowed.
  • Transfer method: Try different transfer methods (Aspera → FTP → Globus) if one fails.
  • Network restrictions: Some corporate networks block certain protocols - try Globus as an alternative.

Java and system requirements

Q: Do I need to install Java separately?
A: No! The tool automatically downloads Java 21 JRE if needed. It checks your system first and only downloads if necessary.

Q: What if the Java download fails?
A: You can install Java 21 manually from Adoptium, or check your internet connection and firewall settings.

Q: Which operating systems are supported?
A: Windows 10+, macOS 10.14+, and Linux. The tool is completely portable and works on all major platforms.

Launcher and startup issues

Q: The tool won't start on macOS - what should I do?
A: macOS security may block the launcher. Right-click start.sh and select "Open", or run xattr -cr . in Terminal.

Q: I get permission errors on Linux/macOS
A: Make the launcher executable by running chmod +x start.sh in Terminal.

Q: The tool says "Another instance is running" but I don't see it
A: Check for background Java processes and kill them, or restart your computer. The tool uses network-based locking.

File management and checksums

Q: What are checksum.txt files?
A: These are automatically generated files that ensure data integrity during upload. They're protected from removal and required for submission.

Q: Can I remove checksum.txt files?
A: No, these files are protected to maintain submission integrity. The tool prevents their removal and regenerates them as needed.

Q: How do I remove individual files?
A: When files are present, a "Remove" column appears with trash icon buttons. Click the trash icon next to any file to remove it individually.
