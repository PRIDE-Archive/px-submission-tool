The File Selection step asks the user to assign correct file types to the files to be submitted. The submission tool does the best to assign file types automatically, however, it is worth double-checking the file types.

Supported File Types

File Type Requirement Description
RESULT Required for complete submissions Experiment result files, either PRIDE XML or mzIdentML are supported.
RAW Required MS instrument raw output such as: BRUKER baf files or non-processed mzML/mzXML files. if your raw outputs are directories, please zip them.
SEARCH Required for partial submission, optional for complete submission (recommend to include) Search engine output or analysis pipeline output files, such as: Mascot .dat, X!Tandem XML files.
PEAK Required when mzIdentML is submitted, optional for partial submission (recommend to include) Processed peak list, such as: mgf, dta, ms2, pkl or processed mzML/mzXML files
QUANT Optional Quantification analysis output files, such as: MaxQuant output file, mzQuantML, etc
GEL Optional Gel image file such as: TIF, JPG, or PNG files
FASTA Optional Sequence database file which contains protein sequences used by the search engine
SPECTRUM_LIBRARY Optional spectra library file used during the analysis
MS_IMAGE_DATA Optional, but required when there are MS image raw files MS image data, such as: imzML
OPTICAL_IMAGE Optional Optical images
OTHER Optional Any other files related to the submission (e.g. scripts)
