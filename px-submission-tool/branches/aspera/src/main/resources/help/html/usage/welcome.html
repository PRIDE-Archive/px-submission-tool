ProteomeXchange Submission Tool assists you to submit your MS/MS datasets to multiple key proteomics data repositories, such as: PRIDE and PeptideAtlas.


Before start the submission, you need to decide on which type of submission to make:
  • Complete submission
  • Partial submission
  • Bulk submission
You can select the type of submission by clicking the buttons highlighted below:

Complete submission

This type is highly recommended. You will be issued with not only a ProteomeXchange accession but also a permanent DOI number to uniquely identify your submission in the future. With additional benefits that, PRIDE will show your data and allow private journal review support. Your data will also be available on the ProteomeXchange FTP to download once it has been made public.

The complete submission requires three sets of information:

  • Result Files: they are either PRIDE XML contains both the spectra and the identifications or mzIdentML with spectrum files such as: MGF, mzML, most of the existing search engine outputs can be converted to PRIDE XML using the PRIDE Converter.
  • Raw Files: MS instrument outputs, such as: BRUKER .baf files. If your raw outputs are directories instead of files, please compress them into individual files before upload.
  • PRIDE Login: PRIDE user name and password. All submissions to the ProteomeXchange are private by default, the user name and password will help us to secure your submissions. You can register a new user account at PRIDE

    • Although not required, we would also recommend you to submit the following data files:

  • Search result files: the original output from your search engine or your analysis pipeline, such as: Mascot .dat files. They should contain the peptide/protein identifications.

    • Partial submission

    This type is not recommended, since it will significantly reduce the visibility and reusability of your dataset. You should only choose this option if your search results cannot be converted by the PRIDE Converter. As a result, you will be only issued with a ProteomeXchange accession, and your dataset will only be available on the ProteomeXchange FTP to download.

  • Search result files: the original output from your search engine or your analysis pipeline, such as: Mascot .dat files. They should contain the peptide/protein identifications.
  • Raw Files: MS instrument outputs, such as: BRUKER .baf files. If your raw outputs are directories instead of files, please compress them into individual files before upload.
  • PRIDE Login: PRIDE user name and password. All submissions to the ProteomeXchange are private by default, the user name and password will help us to secure your submissions. You can register a new user account at PRIDE

    • Bulk submission

    Bulk submission is mainly for submissions that contain a large set of data files, for example, more than 200 files. It helps to avoid manually assigning all the required information. Both complete submissions and partial submissions can be made used through option.

    The bulk submission requires three set of information:

  • Experiment data files: The files you want to submit to the ProteomeXchange, such as: raw instrument outputs, search engine outputs.
  • Submission summary file: This file captures the descriptive information about a ProteomeXchange submission, such as: experimental metadata, file relations, and etc. For the details, please refer to the file format.
